2\. Incorrect pixel settings

- The more pixels you install here, the slower it gets. Be selective with pixels.

- If a pixel doesn't need to be installed, don't.

- Fewer is better (in terms of speed).

![Faq Lemot 4 Min](https://tutorial.scalev.id/wp-content/uploads/2024/09/faq-lemot-4-min-570x1024.png)

3\. Incorrect image settings

- Converting images before uploading. No need to convert images from jpg to webp. Images are automatically converted by Scalev to be optimal.

- See the FAQ tutorial for full-image landing page settings.

4\. Imported from Berdu

- When importing landing pages from Berdu, image files still reference Berdu's server.

- To maximize performance, save the images and re-upload them to move to Scalev's server.

5\. Editing the landing page using Safari (Mac)

- Using Safari causes uploaded jpg images in Scalev not to convert to webp (Apple's default setting). jpg files are heavier than webp.

- Solution if already uploaded via Safari: delete the component, re-add it, and re-upload the image using **Chrome.**

6\. Wrong component priority/placement

Image components generally slow down speed.

If the landing page must be 100% images, split images into 2-3 parts so Scalev loads them incrementally.

Prioritize the first screen (ATF/Above The Fold) to contain 60-70% text components, the rest images.
