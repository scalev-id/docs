Default events are the built-in pixel events automatically configured by Scalev in Landing Page Studio.

Scalev has four default events:

1. ViewContent

3. AddToCart

5. InitiateCheckout

7. Purchase

## Event ViewContent

ViewContent is automatically installed when you first create any new landing page in Scalev.

You can find it at Landing Page Studio → Settings → Analytics Settings

## Event AddToCart

AddToCart is automatically installed when you create a new landing page of the Checkout Page type.

To confirm the AddToCart event is automatically installed, check your landing page settings.

Check under Settings → Analytics Setting.

Scroll down to find it.

<figure>

![Default Events 8](https://tutorial.scalev.id/wp-content/uploads/2024/07/default-events-8.png)

<figcaption>

AddToCart event settings in Scalev landing page

</figcaption>

</figure>

Facebook Pixel ID will be filled in automatically if you've already added an FB pixel in your Scalev Analytics settings.

The Event When Landing Page Opens field will be automatically filled with AddToCart when you create a new landing page of the Checkout Page type.

## Event InitiateCheckout

InitiateCheckout is automatically installed when you add an Order Form component in Landing Page Studio.

When you add this component, you need to assign which Product or Bundle is placed in it.

After assigning the product, you can check the InitiateCheckout event in the Order Form component settings.

Default Purchase event settings for Bank Transfer & E-Payment methods:

- Transfer proof uploaded by the buyer or admin (bank transfer only)

- Transfer proof uploaded by admin only (bank transfer only)

- Order marked as Confirmed

Default Purchase event settings for COD payments:

- Order marked as Confirmed

Those are the Scalev default events installed automatically.

You can change these default Scalev settings yourself.

Pixel events beyond the ones discussed here — like AddToCart, AddPaymentInfo, Lead, etc. — can be added manually to match your business flow.

If these default events already meet your needs, there's no need to change anything.
