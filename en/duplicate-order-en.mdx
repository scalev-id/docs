If you encounter this error during order testing, the cause is:

Order submissions too close together.

Usually it happens because you previously did a test order.

With identical order details.

Either the same name or the same phone number.

So it's detected as a spam order.

That's a preventive step so you don't get hit with spam orders.

**Solution:**

Just wait a few minutes, then you can test the order again.

Or order with different details — different name or different phone number.

Don't forget to also check the [Order Validation](https://tutorial.scalev.id/anti-spam-order/ "Anti Spam Order") setting in your store like this:

<figure>

![Duplicate order 1](https://tutorial.scalev.id/wp-content/uploads/2025/08/duplicate-order-1.png)

Make sure the settings in the red box match your store's needs.

- **Max Number of Orders per Phone Number in One Hour** is the order attempt limit per phone number within 1 hour.

- **Max Number of Orders per IP Address in One Hour** is the order attempt limit per IP address within 1 hour.

The order attempt limit per hour can be adjusted to prevent spam orders.

Including when doing test orders.
