Previously, Scalev could be described as "selfish" when interacting with other tools or software:

- Scalev → Other tool: Can trigger action ✅

- Other tool → Scalev: Cannot trigger action ❌

Like someone who wants to be heard but won't listen.

## Current Solution

Scalev has now become a more collaborative platform:

- Willing to receive input from other tools ✅

- Willing to send data to other tools ✅

- Ready to integrate with any platform ✅

## Integration Steps for End Users

If you use a specific platform/application and want to integrate with Scalev:

### 1\. Request from your platform

Ask them to develop an integration app with Scalev.

### 2\. Wait for implementation

Depends on the platform's willingness to allocate resources for the implementation.

### 3\. Connect in Scalev

Once implemented, both platforms can share data with each other.

Access the platform in the **Settings** > **Developers** > **Apps** menu to complete the integration.

## Guide for Developers

For developers who want to integrate their platform with Scalev:

### 1\. Study the API Documentation

Link: [https://docs.scalev.id/auth.html](https://docs.scalev.id/auth.html)

### 2\. Technical Consultation

Join the Scalev Telegram developer group at "**Coding Area!**": [https://developers.scalev.id/](https://developers.scalev.id/ "https://developers.scalev.id/")

### 3\. Implementation

If you're a competent developer, the two resources above are sufficient to understand how to implement it.

## Conclusion

Scalev now provides integration opportunities for third-party applications via the Apps feature accessible in the dashboard.

Documentation and support channels are available.

Next it's up to third-party platforms to decide whether to take advantage of this integration opportunity.
