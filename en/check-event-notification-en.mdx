---
To find out whether events from Scalev have been sent to Meta Ads, follow the steps below:

1\. Log in to the Scalev dashboard

2\. Open the Order page → Go to the icons on the top right → Click the three-column icon

<figure>

![Cek Notifikasi Event 1](https://tutorial.scalev.id/wp-content/uploads/2024/07/cek-notifikasi-event-1-1024x468.png)

<figcaption>

Click the three-column icon on the top right

</figcaption>

</figure>

3\. The "Configure Order List Columns" dialog appears — check Purchase Event Triggered

<figure>

![Cek Notifikasi Event 2](https://tutorial.scalev.id/wp-content/uploads/2024/07/cek-notifikasi-event-2-1024x455.png)

<figcaption>

Check Purchase Event Trigger then click Save

</figcaption>

</figure>

4\. Once saved, a new "Purchase Event Triggered" column appears on the Order page

<figure>

![Cek Notifikasi Event 3](https://tutorial.scalev.id/wp-content/uploads/2024/07/cek-notifikasi-event-3-1024x566.png)

<figcaption>

Purchase Event Trigger column appears

</figcaption>

</figure>

Notes:

- If the X circle icon to the left of Meta CAPI is gray, the Purchase event hasn't been sent to Meta

- If the checkmark circle icon to the left of Meta CAPI is green, the Purchase event has been sent to Meta

Why is the X circle still gray?

Possible reasons:

- The Purchase event sending is waiting for the server sync cycle between Scalev and Meta

- Order status isn't yet Confirmed / Unpaid, or

- Other order statuses you've configured that keep the Purchase event from triggering
