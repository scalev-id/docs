---
title: "Runtime HTML Mode"
description: "Gunakan method runtime window.Scalev yang tersedia di halaman Scalev HTML Mode."
---

Halaman Scalev HTML Mode dapat memakai `window.Scalev` dari JavaScript browser. Runtime ini memberi akses ke konteks halaman yang aman, Store Context terpilih, helper checkout, analytics forwarding, helper prefill form, dan diagnostics sambil menjaga token API privat tetap di sisi server.

Halaman ini ditulis untuk developer dan AI agent yang membuat file HTML Mode. Jika kamu membuat file HTML untuk Scalev, gunakan referensi ini dan method `window.Scalev` yang terdokumentasi sebagai pengganti URL privat Scalev.

## Ketersediaan

Runtime tersedia di halaman HTML Mode yang sudah dirender. Di preview editor lokal, beberapa method dapat mengembalikan data preview.

Selalu cek runtime sebelum dipakai:

```js
const scalev = window.Scalev;

if (!scalev) {
  // Render fallback content atau pastikan halaman tetap bisa digunakan.
}
```

Sebagian besar method bersifat asynchronous dan mengembalikan `Promise`. `Scalev.data.get()` dan `Scalev.diagnostics.get()` bersifat synchronous.

## Store Context

Halaman HTML Mode bisa memiliki Store Context.

Jika Store Context dipilih, `Scalev.data.get().store` berisi ringkasan store, produk terpilih, dan bundle price options terpilih saja. Perlakukan data ini sebagai katalog checkout khusus halaman.

Jika HTML Checkout Page dipilih, `Scalev.data.get().after_checkout` berisi type aksi setelah checkout berhasil beserta konfigurasi tujuan publiknya. Object ini adalah state konfigurasi dari editor. Runtime mengeksposnya agar halaman kamu bisa menjalankan redirect setelah order dibuat.

Jika Store Context belum dipilih, perlakukan `store` sebagai data yang belum tersedia. Perlakukan halaman sebagai landing page biasa dan gunakan hanya method aman seperti `Scalev.data.get()`, `Scalev.analytics.track()`, dan `Scalev.prefill.get()`.

## Aturan keamanan

- Jaga API key, business-user JWT, storefront API key, customer token, cookie, dan kredensial tetap di luar JavaScript halaman.
- Gunakan method `window.Scalev` yang terdokumentasi sebagai pengganti URL privat Scalev atau private API.
- Gunakan hanya method `window.Scalev` yang terdokumentasi.
- Buat order hanya setelah pengunjung sengaja submit form.
- Utamakan `Scalev.checkout.createOrder()` saat submit; `validateOrder()` bersifat opsional karena `createOrder()` sudah memvalidasi payload.
- Gunakan hanya produk, bundle, harga, stok, payment method, opsi pengiriman, dan identitas customer yang tersedia di data runtime.

## `Scalev.data.get()`

Mengembalikan data publik halaman yang diinject ke halaman.

```js
const data = Scalev.data.get();
```

Bentuk respons:

```json
    "id": 1,
    "uniqueId": "page_uid",
    "username": "brand"
  },
  "store": null,
  "after_checkout": null
```


Gunakan data ini untuk render produk terpilih, bundle option, payment methods aktif, dan form yang memakai store. Variant produk terpilih tersedia di `store.products[].variants[]`. Perlakukan payload ini sebagai seluruh konteks yang tersedia untuk halaman.

`store.payment_method_options` berisi object payment option final yang bisa dipilih. Render `display` sebagai label untuk visitor, gunakan `logo_url` sebagai image source payment, dan submit `value` sebagai `paymentMethod` atau `payment_method`.

Value payment option sudah dibuat flat:

- `va:BRI` berarti virtual account BRI.
- `bt:BCA:payment_account_uid` berarti bank transfer manual untuk account BCA yang diwakili `payment_account_uid`.
- Value regular seperti `cod`, `qris`, `invoice`, `alfamart`, atau `gopay` tetap sama.

Display payment option sudah siap untuk UI client:

- `va:BRI` tampil sebagai `BRI Virtual Account`.
- `va:MANDIRI` tampil sebagai `Bank Mandiri Virtual Account`.
- `va:SAHABAT_SAMPOERNA` tampil sebagai `Bank Sahabat Sampoerna Virtual Account`.
- `qris` tampil sebagai `QRIS`.
- `gopay` tampil sebagai `GoPay`.
- `bt:BCA:payment_account_uid` tampil sebagai `Bank Central Asia (PT Interna Cipta Asia)` saat account holder adalah `PT Interna Cipta Asia`.

Logo payment option sudah siap untuk UI client:

- `va:BRI` memakai `https://cdn.scalev.com/icons/BRI.png`.
- `bt:BCA:payment_account_uid` memakai `https://cdn.scalev.com/icons/BT_BCA.png`.
- `qris` memakai `https://cdn.scalev.com/icons/qris.png`.

Field legacy yang deprecated masih diekspos untuk halaman lama:

- `store.payment_methods`
- `store.sub_payment_methods`
- `store.payment_accounts`

Gunakan hanya `store.payment_method_options` untuk development checkout HTML Mode baru. Perlakukan field deprecated sebagai data referensi legacy untuk halaman lama.

```js
const paymentMethodOptions = store.payment_method_options || [];
const selectedPaymentOption = paymentMethodOptions[0];

const label = selectedPaymentOption.display;
const logoUrl = selectedPaymentOption.logo_url;

const payload = {
  paymentMethod: selectedPaymentOption.value,
};
```

### Konfigurasi aksi setelah checkout

`after_checkout` tersedia di HTML Checkout Page. Gunakan data ini setelah `Scalev.checkout.createOrder()` berhasil.

| Field | Keterangan |
| --- | --- |
| `type` | Salah satu dari `success_page`, `direct_to_whatsapp`, `direct_to_custom_whatsapp`, `other_page`, `order_page`, atau `custom_url`. |
| `custom_whatsapp_phone` | Nomor WhatsApp tetap untuk `direct_to_custom_whatsapp`, jika dikonfigurasi. |
| `other_page_path` | Path dari slug landing page terpilih, misalnya `/thank-you`. |
| `custom_url` | URL tujuan untuk `custom_url`, jika dikonfigurasi. |
| `handler_assignment` | Mode assignment sales store terpilih untuk `direct_to_whatsapp`, misalnya `rotator` atau `fixed`. |

Halaman kamu menjalankan aksi ini setelah order dibuat. Ikuti [Aksi setelah order HTML Mode](/id/aksi-setelah-order-html-mode) dan gunakan state konfigurasi ini untuk memilih redirect. Untuk `direct_to_whatsapp`, sales person terpilih diselesaikan saat order dibuat; gunakan `handler_phone` dari response order.

## Method lokasi

Gunakan method ini untuk form alamat dan pengiriman.

```js
const provinces = await Scalev.location.provinces();
const cities = await Scalev.location.cities(provinceId);
const subdistricts = await Scalev.location.subdistricts(cityId);
const postalCodes = await Scalev.location.postalCodes(subdistrictId);
```

`provinces`, `cities`, dan `subdistricts` menerima query object opsional:

```js
const provinces = await Scalev.location.provinces({ q: "Jawa" });
const cities = await Scalev.location.cities(provinceId, { q: "Bandung" });
const subdistricts = await Scalev.location.subdistricts(cityId, {
  q: "Coblong",
});
```

Gunakan ID dari step sebelumnya. Gunakan hardcoded location ID hanya saat user memberikannya secara eksplisit.

## `Scalev.checkout.validateDiscount(payload)`

Memvalidasi kode diskon untuk payload checkout.

```js
const store = Scalev.data.get().store;

const result = await Scalev.checkout.validateDiscount({
  code: "PROMO10",
  grossRevenue: 179000,
  netProductPrice: 179000,
  shippingCost: 0,
  paymentMethod: store.payment_method_options?.[0]?.value || "cod",
});
```

Nama field dapat memakai camelCase atau snake_case:

- `code`
- `grossRevenue` atau `gross_revenue`
- `netProductPrice` atau `net_product_price`
- `shippingCost` atau `shipping_cost`
- `pageUniqueId` atau `page_unique_id`
- `paymentMethod` atau `payment_method`
- `domain`

`pageUniqueId` otomatis memakai halaman saat ini, dan `domain` otomatis memakai hostname saat ini. Jika value option terpilih adalah value flat seperti `va:BRI` atau `bt:BCA:payment_account_uid`, kirim value itu apa adanya sebagai `paymentMethod`. Runtime akan menormalisasinya sebelum memanggil API Scalev.

Gunakan hanya untuk flow checkout yang terhubung dengan store.

## `Scalev.checkout.searchWarehouses(payload)`

Mengembalikan opsi warehouse untuk variant dan tujuan terpilih.

```js
const data = Scalev.data.get();
const firstVariant = data.store.products[0].variants[0];

const warehouses = await Scalev.checkout.searchWarehouses({
  destinationSubdistrictId: selectedSubdistrictId,
  ordervariants: [{ variant_unique_id: firstVariant.unique_id, quantity: 1 }],
});
```

Nama field yang diterima:

- `storeId`, `store_id`, atau `store`; tanpa nilai store, runtime memakai Store Context terpilih.
- `destinationSubdistrictId`, `destination_subdistrict_id`, `subdistrictId`, `subdistrict_id`, `destinationId`, `destination_id`, `destination`, `locationId`, `location_id`, atau `location`
- `variants`, `orderVariants`, atau `ordervariants`: array `{ variantId | variant_id | id | variant_unique_id | variantUniqueId | unique_id | uniqueId, qty | quantity }`
- `orderId`, `order_id`, atau `order` untuk flow order yang sudah ada

Gunakan variant terpilih dari `Scalev.data.get().store.products`. Kamu boleh mengirim value publik `variant_unique_id`; runtime akan mencocokkannya ke numeric `variant_id` yang dibutuhkan endpoint warehouse landing page lama.

## `Scalev.checkout.searchCouriers(payload)`

Mengembalikan opsi courier service untuk store, tujuan, warehouse, dan payment method.

```js
const firstWarehouse = warehouses.data?.[0] || warehouses[0];

const couriers = await Scalev.checkout.searchCouriers({
  paymentMethod: "cod",
  destinationSubdistrictId: selectedSubdistrictId,
  warehouse: firstWarehouse,
  weight: 1200,
});
```

Nama field yang diterima:

- `storeId`, `store_id`, atau `store`; tanpa nilai store, runtime memakai Store Context terpilih.
- `paymentMethod` atau `payment_method`
- `destinationSubdistrictId`, `destination_subdistrict_id`, `subdistrictId`, `subdistrict_id`, `locationId`, `location_id`, `destinationId`, `destination_id`, `destination`, atau `location`
- `warehouseId`, `warehouse_id`, atau `warehouse`. Kamu bisa mengirim object warehouse dari hasil `searchWarehouses`.
- `weight`

Gunakan method ini setelah pengunjung memilih produk dan tujuan pengiriman. Backend membutuhkan numeric `warehouse_id`; runtime mengambil id itu dari object warehouse jika memungkinkan.

Opsi courier yang dikembalikan berisi `logo_url` saat courier code tersedia. Gunakan sebagai image source courier. Contoh: Ninja Xpress mengembalikan `https://cdn.scalev.com/assets/images/kurir/ninja.png`.

## `Scalev.checkout.validateOrder(payload)`

Opsional untuk memvalidasi bentuk payload order sebagai preflight check. Helper ini berguna untuk debugging. Pada flow submit normal, panggil `createOrder` dan tangani error validasinya.

```js
const data = Scalev.data.get();
const store = data.store;
const firstVariant = store.products[0].variants[0];

const validation = await Scalev.checkout.validateOrder({
  page: data.page.uniqueId,
  store: store.unique_id,
  customerName: "Customer Name",
  customerPhone: "08123456789",
  customerEmail: "customer@example.com",
  address: "Customer address",
  location: 12345,
  ordervariants: [{ variant_unique_id: firstVariant.unique_id, quantity: 1 }],
  paymentMethod: "cod",
});
```

Bentuk respons:

```json
{
  "valid": true,
  "normalized_payload": {
    "page_unique_id": "page_uid",
    "store_unique_id": "store_uid",
    "customer_name": "Customer Name",
    "customer_phone": "08123456789"
  },
  "warnings": []
}
```

Jika `valid` bernilai `false`, baca `warnings` dan perbaiki payload form sebelum membuat order.

Warning umum mencakup page id, store id, nama customer, nomor customer, atau item order yang belum ada.

## `Scalev.checkout.createOrder(payload)`

Membuat public order sungguhan dan memvalidasi payload. Panggil method ini hanya setelah pengunjung sengaja submit form. `validateOrder` bersifat opsional sebagai preflight helper.

```js
async function submitOrder() {
  const data = Scalev.data.get();
  const store = data.store;
  const firstVariant = store.products[0].variants[0];
  const selectedPaymentOption = store.payment_method_options[0];

  const payload = {
    page: data.page.uniqueId,
    store: store.unique_id,
    customerName: "Customer Name",
    customerPhone: "08123456789",
    customerEmail: "customer@example.com",
    address: "Customer address",
    location: 12345,
    ordervariants: [{ variant_unique_id: firstVariant.unique_id, quantity: 1 }],
    paymentMethod: selectedPaymentOption.value,
  };

  const order = await Scalev.checkout.createOrder(payload);
  return order;
}
```

`createOrder` menerima key payload publik yang sama dengan flow checkout regular editor Scalev. Runtime menormalisasi payload sebelum mengirim request order:

- Customer fields: `customerName`, `customerPhone`, `customerEmail`, `name`, `phone`, `address`
- Store dan page fields: `store`, `page`, plus alias seperti `storeUniqueId`, `store_unique_id`, `pageUniqueId`, dan `page_unique_id`
- Lokasi dan shipping fields: `location`, `destinationSubdistrictId`, `subdistrictId`, `postalCode`, `shippingOrigin`, `courierService`, `shippingCost`, dan `courierAggregator`
- Payment field: `paymentMethod` atau `payment_method`
- Item fields: `ordervariants` dengan `{ variant_unique_id, quantity }`, `orderbundles` dengan `{ bundle_price_option_unique_id, quantity }`, atau legacy `orderlines`
- Diskon dan pricing fields: `discountCodeCode`, `productDiscount`, `shippingCost`, `shippingDiscount`

Gunakan value `unique_id` dari `Scalev.data.get().store` sebagai identifier item order.

Untuk pilihan shipping, kirim object warehouse terpilih sebagai `shippingOrigin` atau kirim `unique_id` warehouse sebagai `warehouseUniqueId`. Kirim object courier service terpilih sebagai `courierService` atau kirim `id` service sebagai `courierServiceId`. Render image courier dari `logo_url` option `searchCouriers()`. Runtime mengirim `warehouse_unique_id` dan `courier_service_id` dalam shape backend.

Untuk pilihan payment, render `store.payment_method_options[].display`, render image payment dari `store.payment_method_options[].logo_url`, dan submit hanya `value` option terpilih. Runtime mengubah `va:BRI` menjadi `payment_method: "va"` plus `sub_payment_method: "BRI"` untuk backend, dan mengubah `bt:BCA:payment_account_uid` menjadi `payment_method: "bank_transfer"` plus `payment_account_unique_id: "payment_account_uid"`.

Buat order hanya dari aksi submit pengunjung.

Setelah `createOrder` berhasil, HTML Checkout Page otomatis men-trigger event analytics submit form yang sudah dikonfigurasi di sisi client dan server. Event hanya dikirim untuk pixel analytics yang sudah dikonfigurasi pada page. HTML Checkout Page baru di editor sudah terisi awal dengan Meta dan TikTok `InitiateCheckout`, SnackVideo client `formSubmit`, dan SnackVideo server `EVENT_FORM_SUBMIT`; kamu bisa mengubahnya sebelum menyimpan page. Tambahkan event analytics terpisah hanya saat memang perlu event custom tambahan.

Pilih sendiri jenis redirect setelah order. Runtime membuat order dan men-trigger submit analytics; halaman kamu menangani navigasi. Baca `Scalev.data.get().after_checkout` jika tersedia, lalu ikuti [Aksi setelah order HTML Mode](/id/aksi-setelah-order-html-mode).

Data order yang dikembalikan adalah public response ringkas untuk redirect dan UI payment buyer-facing. Gunakan `secret_slug`, `public_order_url`, `payment_url`, `handler_phone`, `chat_message`, total, line item, field payment, serta object map `variants` dan `bundle_price_options` yang sudah ada. ID order internal, revenue khusus dashboard, platform fee, dan atribusi affiliate berada di luar kontrak runtime publik.

### Error handling

Method runtime me-reject `Promise` dengan JavaScript `Error`. Gunakan JavaScript `Error` sebagai surface error customer-facing yang stabil.

Gunakan `error.message` untuk copy UI. Gunakan `error.status`, `error.data`, dan `Scalev.diagnostics.get()` hanya untuk debugging atau handling lanjutan. Jaga raw `error.data.error` tetap di luar UI customer-facing karena kegagalan upstream bisa mengembalikan string, boolean, object, atau map error per field.

```js
try {
  const order = await Scalev.checkout.createOrder(payload);
  return order;
} catch (error) {
  showFormError(error.message || "Order gagal dibuat.");
  console.debug("Scalev checkout error", {
    status: error.status,
    data: error.data,
    diagnostics: Scalev.diagnostics.get(),
  });
}
```

Gunakan method `Scalev.checkout.*` sebagai pengganti URL proxy runtime `/api/v2/...` supaya error validasi dinormalisasi sebelum sampai ke UI.

## `Scalev.analytics.track(providerOrPayload, payload?)`

Meneruskan event analytics yang sudah dikonfigurasi melalui Scalev.

```js
await Scalev.analytics.track("facebook", {
  firstName: "Customer",
  phone: "08123456789",
  variants: [{ unique_id: "variant_uid", quantity: 1, price: 179000 }],
  events: [
    {
      event_name: "Lead",
      parameters: { value: 179000, currency: "IDR" },
    },
  ],
});
```

Kamu juga bisa mengirim satu object:

```js
await Scalev.analytics.track({
  provider: "tiktok",
  payload: {
    events: [
      {
        event: "CompletePayment",
        parameters: { value: 179000, currency: "IDR" },
      },
    ],
  },
});
```

Nama provider yang didukung:

- `fb`
- `facebook`
- `tiktok`
- `kwai`

Runtime meneruskan shape payload analytics Scalev lama untuk setiap provider. Runtime menambahkan page id saat ini, event source URL, referrer, dan attribution aman dari cookie browser atau URL parameter saat tersedia. Untuk event Facebook dan TikTok, `event_id` dibuat otomatis saat value belum dikirim.

Untuk atribusi produk, kirim `variants`, `orderVariants`, atau `ordervariants` dengan `variant_unique_id` / `unique_id`, dan `bundlePriceOptions`, `bundle_price_options`, atau `orderbundles` dengan `bundle_price_option_unique_id` / `unique_id`.

Track hanya event yang sesuai dengan interaksi halaman. Jaga kredensial dan customer token privat tetap di luar payload analytics.

## `Scalev.prefill.get()`

Membaca data prefill form yang aman dan tersimpan di browser.

```js
const prefill = await Scalev.prefill.get();
const formData = prefill.data || {};
```

Bentuk respons:

```json
{
  "data": {
    "customerName": "Customer Name",
    "customerPhone": "08123456789"
  }
}
```

Gunakan ini untuk mengisi ulang field form. Anggap semua value opsional.

## `Scalev.prefill.save(form, metadata?)`

Menyimpan data prefill form yang aman untuk kunjungan berikutnya.

```js
await Scalev.prefill.save(
  {
    customerName: "Customer Name",
    customerPhone: "08123456789",
  },
  {
    source: "html_mode",
  },
);
```

Simpan hanya value form biasa yang aman untuk diingat. Jaga password, token, rahasia pembayaran, dan kredensial privat tetap di luar prefill storage.

## `Scalev.customer.prefill()`

Mengembalikan profil customer termediasi saat pengunjung memiliki sesi customer storefront aktif. Fallback result method ini adalah `null`.

```js
const customer = await Scalev.customer.prefill();

if (customer) {
  // Gunakan data customer mediated.
}
```

Runtime memanggil endpoint lokal Scalev supaya token customer tetap berada di server. Selalu handle `null`.

## `Scalev.diagnostics.get()`

Mengembalikan diagnostics runtime terbaru untuk debugging.

```js
const diagnostics = Scalev.diagnostics.get();
```

Diagnostics dapat mencakup request runtime yang gagal dan, saat `?agent_debug=1` aktif, warning visual seperti horizontal overflow atau form control yang terlihat tetapi belum memiliki label. Gunakan diagnostics hanya untuk debugging dan QA.

## Contoh minimal saat store belum tersedia

Gunakan pola ini saat Store Context belum dipilih:

```html
<script>
  async function init() {
    const scalev = window.Scalev;
    const data = scalev?.data?.get
      ? scalev.data.get()
      : { page: {}, store: null };
    const prefill = scalev?.prefill?.get
      ? await scalev.prefill.get()
      : { data: {} };

    document.querySelector("[name='name']").value =
      prefill.data?.customerName || "";

    document.querySelector("form").addEventListener("submit", async (event) => {
      event.preventDefault();
      const form = Object.fromEntries(new FormData(event.currentTarget));
      await scalev?.prefill?.save?.(form, { source: "lead_form" });
    });
  }

  init();
</script>
```

## Contoh minimal checkout store

Gunakan pola ini hanya saat Store Context dipilih:

```html
<script>
  async function submitOrder(event) {
    event.preventDefault();

    const scalev = window.Scalev;
    const data = scalev.data.get();
    const store = data.store;
    const firstProduct = store.products[0];
    const firstVariant = firstProduct.variants[0];

    const payload = {
      page: data.page.uniqueId,
      store: store.unique_id,
      customerName: event.currentTarget.elements.customerName.value,
      customerPhone: event.currentTarget.elements.customerPhone.value,
      ordervariants: [
        { variant_unique_id: firstVariant.unique_id, quantity: 1 },
      ],
      paymentMethod: "cod",
    };

    const order = await scalev.checkout.createOrder(payload);
    console.log(order);
  }
</script>
```
