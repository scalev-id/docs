1\. Salah/kurang melakukan settingan font.

**a. Bagian pengaturan → Matikan custom font**

![Faq Lemot Min](https://tutorial.scalev.id/wp-content/uploads/2024/09/faq-lemot-min.png)

**b. Bagian desain → System default**

![Faq Lemot 2 Min](https://tutorial.scalev.id/wp-content/uploads/2024/09/faq-lemot-2-min.png)

**c. Bagian elemen text → System default**

![Faq Lemot 3 Min](https://tutorial.scalev.id/wp-content/uploads/2024/09/faq-lemot-3-min.png)

2\. Salah Setting pixel

- Semakin banyak pixel yang Anda pasang di sini, akan semakin lemot. Bijak dalam memasang pixel. 

- Jika tidak perlu dipasang misalnya, tidak usah dipasang. 

- Makin sedikit makin baik (dalam konteks speed).

![Faq Lemot 4 Min](https://tutorial.scalev.id/wp-content/uploads/2024/09/faq-lemot-4-min-570x1024.png)

3\. Salah setting image

- Meng-convert image sebelum diupload. Tidak perlu meng-convert image dari jpg menjadi webp. Image akan diconvert secara otomatis oleh Scalev agar optimal.

- Arahkan ke tutorial FAQ settingan LP full image.

4\. Import dari Berdu

- Untuk import LP dari Berdu, file image yang diakses masih menggunakan server Berdu.

- Agar maksimal, maka gambar bisa di-save dulu kemudian diupload ulang agar pindah ke server Scalev.

5\. Mengedit LP menggunakan Browser Safari (Mac)

- Menggunakan browser Safari akan membuat image jpg yang diupload ke dalam Scalev tidak berubah menjadi webp (settingan default Apple). File jpg lebih berat dibandingkan webp.

- Solusi jika sudah diupload melalui browser Safari: hapus komponennya, pasang ulang, upload ulang gambarnya melalui browser **Chrome.**

6\. Salah prioritas/penempatan komponen

Umumnya komponen image akan membuat speed melambat. 

Jika memang harus 100% image, pecah image menjadi 2-3 bagian sehingga loadingnya akan dicicil oleh Scalev

Prioritaskan untuk layar pertama yang diload (ATF/Above The Fold) berisi 60-70% komponen teks, sisanya image.
