Anda mungkin menerima peringatan dari Meta di Events Manager terkait error seperti click ID, client\_ip\_address, fbp, fbc, dll.

Nah, Anda tidak perlu khawatir karena bisa jadi itu disebabkan oleh settingan pixel Anda di masa lalu.

Solusinya: pastikan pixel Anda mengikuti settingan Best Practices dari Meta.

Pertama, Anda harus membuat pixel Dataset dulu.

Caranya:

1. Klik **Settings**
2. Klik menu **Datasets**
3. Klik tombol **Add** 

<Frame>
  ![Buat Conversion Api 1](/images/buat-conversion-api-1.png)
</Frame>

Dikatakan Dataset karena sekarang datangnya data event tidak hanya dari pixel saja, melainkan dari berbagai sumber, bisa dari: pixel, Conversion API, App, atau Offline Conversion.

Untuk pemain toko online umumnya akan membutuhkan 3 data event (kecuali App).

Lanjut.

Klik **Create new dataset**

<Frame>
  ![Buat Conversion Api 2](/images/buat-conversion-api-2.png)
</Frame>

Di popup berikutnya:

<Frame>
  ![Buat Conversion Api 3](/images/buat-conversion-api-3.png)
</Frame>

1. **Name** - beri nama untuk dataset baru Anda
2. Matikan centang jika field kotak kecil ini tercentang
3. Klik tombol **Create**

Setelah berhasil dibuat, pergi ke **Events Manager.**

1. Klik nama dataset baru Anda
2. Klik ikon titik tiga
3. Klik **Open in Events Manage**

<Frame>
  ![Buat Conversion Api 4](/images/buat-conversion-api-4.png)

  Di halaman Events Manager klik tab **Settings** lalu scroll ke bawah untuk mencari bagian **Conversions API**.

  ![Buat Conversion Api 6a](/images/buat-conversion-api-5.png)
</Frame>

Di bagian **Conversions API** \> **Setup direct integration** \> **Set up with Dataset Quality API** \> klik link **Generate access token**

<Frame>
  ![Buat Conversion Api 6b](/images/buat-conversion-api-6a.png)
</Frame>

**Dataset Quality API** adalah update terbaru dari Meta yang katanya berfungsi untuk meningkatkan kinerja iklan menjadi lebih baik dibandingkan dengan yang tanpa ini.

Tapi berdasarkan pengalaman user, hasilnya tidak selalu lebih baik.

Jadi kami sarankan untuk menggunakan versi **non-Quality API**.

**NOTE:** Sekali mengaktifkan Dataset Quality API pada dataset Anda maka Anda **tidak dapat mengembalikannya ke versi lama**. Pertimbangkan terlebih dahulu sebelum membuat keputusan.

<Frame>
  ![Buat Conversion Api 6c](/images/buat-conversion-api-6b.png)
</Frame>

Pastikan Anda memilih dataset yang benar-benar ingin memakai fitur Dataset Quality API ini.

Pilihan yang centangnya sudah berwarna abu artinya sudah aktif dengan fitur ini.

Centang kosong pada dataset yang tidak diinginkan.

<Frame>
  ![15 752x1024](/images/buat-conversion-api-6c.png)
</Frame>

Token inilah yang harus Anda copy lalu masukkan ke pixel Anda di pengaturan Analitik di Scalev.

<Frame>
  ![Business Settings Role 2048x1499](/images/15-752x1024.png)
</Frame>

Ulangi **Generate access token** ini setiap 90 hari supaya kinerja dataset Anda terus berjalan dengan lancar.

**NOTE:**

Sebelum user dapat menggenerate access token, maka pastikan terlebih dulu role user tersebut di BM Anda sudah memiliki akses ke **Apps and integrations**.

Cara memastikannya seperti ini:

1. Masuk ke BM Meta Anda dan klik ikon gear di sebelah kiri bawah

2. Lalu klik menu People, lalu muncul daftar orang yang ada di BM Anda

3. Klik titik tiga di sebelah kanan nama orang yang ingin dipastikan hak aksesnya

<Frame>
  ![Business Settings Role 2048x1499](/images/business-settings-role-2048x1499.jpeg)
</Frame>

4. Akan muncul dialog seperti screenshot ini:

<Frame>
  ![Business Settings Role 2](/images/business-settings-role-2.png)
</Frame>

5. Pastikan toggle **Manage** untuk "Apps and integrations" sudah menyala, kalau belum aktif maka nyalakan dulu.

6. Klik tombol Save

Setelah itu Anda atau anggota di BM Anda dapat menggenerate access token pada pixel yang akan digunakan beriklan.

## Aktifkan fitur Automatic Website Matching

Selanjutnya coba pastikan pengaturan **Automatic Website Matching** di tab Settings pixel Anda diaktifkan semua.

Supaya Meta dapat mencocokkan parameter yang dikirimkan melalui CAPI dari website Anda.

Jika posisi sebelumnya **Off** (nonaktif) seperti ini:

<Frame>
  ![Automatic Website Matching 1](/images/automatic-website-matching-1.png)
</Frame>

Maka klik toggle Off tersebut untuk mengaktifkannya **On** (aktif).

<Frame>
  ![Automatic Website Matching 2](/images/automatic-website-matching-2.png)
</Frame>

## Nonaktifkan fitur tracking otomatis

Masih di halaman tab **Settings** dataset baru Anda, scroll ke bawah dan cari bagian **Events Setup** \> **Track events automatically without code**.

Pastikan fitur ini **Off** (nonaktif) ini untuk mencegah event random yang mungkin tidak akan diperlukan untuk funnel bisnis Anda.
